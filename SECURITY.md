# Security Policy

## Supported version

Security fixes are applied to the latest version on the `main` branch.

## Reporting a vulnerability

Please do not open a public issue for a security vulnerability.

Use GitHub's private vulnerability reporting or private security-advisory feature for this repository when available. If that option is unavailable, contact the maintainer privately through the contact information listed on the maintainer's GitHub profile:

https://github.com/RayyanKhan4004

Include:

- A clear description of the vulnerability
- Steps to reproduce it
- The affected route, component, or dependency
- The possible impact
- A suggested fix, when available

Please allow the maintainer a reasonable opportunity to investigate and release a fix before publicly disclosing the vulnerability.
